IP地址的获取

发表于 2019-09-10 分类于 linux ， dhcp

在Linux最小安装之后，一般需要手动配置网络地址。

一、简单的利用DHCP获取

1、进网卡配置文件

cd /etc/sysconfig/network-scripts/

2、列出network-scripts目录下的文件

ls /etc/sysconfig/network-scripts/

3、找到ifcfg-ens33,进入ifcfg-ens33文件，修改onboot的no为yes

4、重启网络地址：

5.重启或关机后再次连接

二、手动配置IP。

1、进入网卡配置文件

cd /etc/sysconfig/network-scripts/

2、ls列出network-scripts目录下的文件

ls /etc/sysconfig/network-scripts/

3、找到ifcfg-ensXX,比如ifcfg-ens33,配置ifcfg-ens33文件

在第三步中，得到子网和掩码，所用IP可以从192.168.59.3-192.168.59.254中随便获取一个。

4、保存退出，并且重启网络地址。

Systemctl restart network

5.重启或关机后再次连接

如果IP是自己配置的，则说明网络配置成功。

ip -a / ifconfig #查看网卡

ISO七层模型

发表于 2019-09-09 更新于 2019-09-10 分类于 网络

一、OSI七层模型

层数 表示 说明 作用
7 应用层 HTTP、ftp 协议
6 表示层 UTF-8 将应用层协议翻译成计算机可识别的语言
5 会话层 管理传输层
4 传输层 TCP/UDP 建立以及断开连接
3 网络层 IP 1、寻址；2、身份确认
2 数据链路层 MAC物理地址 两设备间的互相通信
1 物理层 传输媒介 将0/1转换成光/电信号

说明：

1、 物理地址（MAC）：
网卡唯一的身份识别，由12位的十六进制数组组成。

（PS：每个设备全世界只能找到一个）

2、数据发送时，是由物理层到应用层逐一打包，而接收时从应用层到物理层层层解封。

二、 IP

1.说明

IP（Internet Protocol）：网络协议

IP地址一般组成范围：

[0254].[ 0254].[ 0254].[ 0254]

2.子网掩码

作用：划分网段。

一般有：

Question：

怎么查看两个网是否处于同一网段？

1) 将IP地址和子网掩码（十进制）转化为二进制；

2) 将转化后的IP地址和子网掩码进行“与”运算；

3) 将运算后的结果转化为十进制；

4) 将两个网转化后的十进制进行对比，看是否相同，相同为同一网段，不同反之。

Eg：

例如，有两台主机，主机一的IP地址为222.21.160.6，子网掩码为255.255.255.192，主机二的IP地址为222.21.160.73，子网掩码为255.255.255.192。现在主机一要给主机二发送数据，先要判断两个主机是否在同一网段。

主机一
222.21.160.6    即：11011110.00010101.10100000.00000110
255.255.255.192 即：11111111.11111111.11111111.11000000
与运算结果为：      11011110.00010101.10100000.00000000
十进制形式为（网络地址）：222.21.160.0

主机二
222.21.160.73   即：11011110.00010101.10100000.01001001
255.255.255.192 即：11111111.11111111.11111111.11000000
与运算结果为：      11011110.00010101.10100000.01000000
十进制形式为（网络地址）：222.21.160.64

博客搭建

发表于 2019-09-09

博客搭建
利用虚拟机在Ubauntu或者deepin 搭建简易网站，利用GitHub.io和hexo.io

0、环境检测

1、系统升级（图形更新）
#update-manager         检测状态

2、检测升级（命令更新）
#sudo apt update 
#sudo apt -y dist-upgrade

一、安装hexo博客本地段

1、安装软件管理工具
#sudo apt -y install npm
如果安装不了，安装替补npm软件管理器
#sudo apt -y install node node-bin

2、通过npm安装hexo
#sudo npm install hexo-cli -g

3、安装下载程序（Git工具包）
#sudo apt -y install git

4、开始部署本地端（hexo）
#sudo hexo init blog（博客文件名可以修改)

出现“安装错误”，补救法：
#cd blog
#sudo npm install

注意：安装过程卡顿时间过长“Ctrl+C”停止，从第1步再试一次，请多次尝试。

二、运行测试环境

1、进入hexo目录
#sudo hexo g

2、初始化hexo
#cd blog(博客文件名)

3、运行本地服务端
#sudo hexo s

4、测试访问
#sudo firefox localhost:4000
x-special/nautilus-clipboard

注意等第3步完成后，在虚拟机Google浏览器或者Firefox浏览器网址输入端输入：localhost:4000，若出现以下界面，则本地端搭建成功。
